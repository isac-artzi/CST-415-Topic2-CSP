What is a Constraint Satisfaction Problem?

A Constraint Satisfaction Problem (CSP) is a mathematical problem defined by:

Variables

X = {X₁, X₂, ..., Xₙ}

Set of variables we need to assign values to

Domains

D = {D₁, D₂, ..., Dₙ}

Possible values for each variable

Constraints

C = {C₁, C₂, ..., Cₘ}

Rules that limit combinations of values

Goal: Find an assignment of values to variables that satisfies all constraints

Complete Assignment: Every variable has a value
Consistent Assignment: No constraints are violated
Solution: Complete + Consistent

Classic Example: Map Coloring

Problem:

Color a map so that no two adjacent regions have the same color

CSP Formulation:

  • Variables: Regions on the map (WA, NT, SA, Q, NSW, V, T)
  • Domains: {Red, Green, Blue} for each region
  • Constraints: Adjacent regions must have different colors
Example Constraints:
• WA ≠ NT
• WA ≠ SA
• NT ≠ SA
• NT ≠ Q
• ...

Why CSP? This is more efficient than searching through all 3⁷ = 2,187 possible color combinations!

Game Example: N-Queens Problem

Problem:

Place N queens on an N×N chessboard so that no two queens attack each other

CSP Formulation:

  • Variables: Qᵢ = column position of queen in row i
  • Domains: {1, 2, 3, ..., N} for each queen
  • Constraints:
    • No two queens in same column: Qᵢ ≠ Qⱼ
    • No two queens on same diagonal: |Qᵢ - Qⱼ| ≠ |i - j|

Lab Question: Knights on Chessboard

Problem:

Place k knights on an n×n chessboard such that no two knights are attacking each other

CSP Formulation:

Variables: K₁, K₂, ..., Kₖ (each knight's position)

Domain: All squares on the n×n board
D = {(i,j) | 1 ≤ i,j ≤ n}

Constraints: For any two knights Kᵢ and Kⱼ:
• Kᵢ ≠ Kⱼ (different squares)
• |xᵢ - xⱼ| + |yᵢ - yⱼ| ≠ 3 or |(xᵢ - xⱼ) × (yᵢ - yⱼ)| ≠ 2
  (not in L-shape attack pattern)

Knight Attack Pattern: Knights attack in an L-shape: 2 squares in one direction, 1 square perpendicular

Backtracking Search Algorithm

The fundamental algorithm for solving CSPs

function BACKTRACKING-SEARCH(csp) returns solution or failure return BACKTRACK({}, csp) function BACKTRACK(assignment, csp) returns solution or failure if assignment is complete then return assignment var ← SELECT-UNASSIGNED-VARIABLE(csp) for each value in ORDER-DOMAIN-VALUES(var, assignment, csp) do if value is consistent with assignment then add {var = value} to assignment inferences ← INFERENCE(csp, var, value) if inferences ≠ failure then add inferences to assignment result ← BACKTRACK(assignment, csp) if result ≠ failure then return result remove {var = value} and inferences from assignment return failure

Key Idea: Incrementally build assignments, backtrack when constraints are violated

Heuristic 1: Most Constrained Variable

Also called: Minimum Remaining Values (MRV) or "fail-first"

Heuristic: Choose the variable with the fewest legal values remaining

MRV(X) = argmin |Domain(X)|

Why does this work?

  • Fail-first: Detect failures early in the search tree
  • Pruning: Reduce the branching factor
  • Efficiency: Avoid wasting time on dead-end paths

Example in Game:

In Sudoku, choose the cell with the fewest possible numbers first

If a cell can only be 3 or 7, fill it before a cell that could be 1, 2, 3, 4, 5, 6, 7, 8, or 9

Heuristic 2: Least Constraining Value

Choose the value that rules out the fewest values in the remaining variables

Heuristic: Prefer the value that leaves the most flexibility for future assignments

LCV(v) = argmin Σ (constraints eliminated by assigning v)

Why Most Constrained Variable + Least Constraining Value?

Variable Selection

Most Constrained

Fail fast - detect problems early

Value Selection

Least Constraining

Succeed quickly - preserve options

Strategy: Choose the hardest variable first, but give it the easiest value

Inference: Forward Checking

Keep track of remaining legal values for unassigned variables and eliminate inconsistent values

function FORWARD-CHECKING(csp, var, value) returns inferences or failure inferences ← {} for each unassigned variable Y that is connected to var by constraint do for each value v in Domain(Y) do if v conflicts with value assigned to var then remove v from Domain(Y) add {Y ≠ v} to inferences if Domain(Y) is empty then return failure return inferences

Example:

WA = red
↓ Forward Checking
NT ∈ {green, blue} (remove red)
SA ∈ {green, blue} (remove red)

Arc Consistency (AC-3 Algorithm)

A stronger form of constraint propagation

Definition:

Arc (X, Y) is consistent if:
For every value x in Domain(X),
there exists some value y in Domain(Y)
that satisfies the constraint between X and Y
function AC-3(csp) returns false if inconsistency found, true otherwise queue ← all arcs in csp while queue is not empty do (X, Y) ← REMOVE-FIRST(queue) if REVISE(csp, X, Y) then if size of Domain(X) = 0 then return false for each Z in neighbors(X) except Y do add (Z, X) to queue return true function REVISE(csp, X, Y) returns true iff Domain(X) was revised revised ← false for each x in Domain(X) do if no value y in Domain(Y) satisfies constraint(X,Y) then delete x from Domain(X) revised ← true return revised

Example: Dungeon Room Placement

Variables: Each room R₁, R₂, ..., Rₙ

Domains: Grid positions (x, y)

Constraints:

  • Rooms must not overlap
  • Rooms must be connected by paths
  • Boss room must be furthest from start
  • Treasure rooms must be in dead ends

Summary & Key Takeaways

Core Concepts:

  • CSP Components: Variables, Domains, Constraints
  • Backtracking: Systematic search with backtracking
  • Heuristics: Most constrained variable + Least constraining value
  • Inference: Forward checking and arc consistency (AC-3)
Time Complexity:
Worst case: O(d^n) where d = domain size, n = number of variables
With heuristics and inference: Often dramatically better in practice

For Your Project:

  1. Identify the CSP in your game/simulation
  2. Formulate variables, domains, and constraints clearly
  3. Implement backtracking with MRV and LCV heuristics
  4. Use forward checking or AC-3 for efficiency
  5. Test with objective functions to measure solution quality
